# 本地关键词备注网页服务设计

## 1. 背景与目标
用户需要一个本地网页服务，用于存储和检索“关键词 -> 备注内容（文本 + 图片）”的数据。

目标能力：
- 通过主关键词搜索并命中条目。
- 通过条目正文中的任意关键词搜索并命中条目。
- 条目支持可选头像图标，搜索结果中可展示该图标。
- 全部使用本地存储，不依赖需要独立运行的数据库服务。
- 页面保持简约，优先可用性。

约束与范围：
- 运行环境：本地 Node.js。
- 存储方案：SQLite 单文件 + 本地文件目录（图片/图标）。
- 数据规模预期：< 200 条。
- 本期不做导入导出备份。

## 2. 方案选择
已讨论方案：
1. Node.js + Express + JSON 文件。
2. Node.js + SQLite 单文件。
3. 纯前端 + IndexedDB。

最终选择：**方案 2（Node.js + SQLite 单文件）**。

选择理由：
- 仍满足“无需独立数据库服务”。
- 比 JSON 文件更利于后续查询和数据一致性维护。
- 对当前小规模数据足够轻量。

## 3. 系统架构
### 3.1 组成
- 本地 Web 服务：Express。
- 本地数据库文件：`data/app.db`（SQLite）。
- 本地文件存储：
  - `data/icons/`（条目头像图标）
  - `data/images/`（条目正文图片）
- 前端：极简单页 HTML（搜索 + 列表 + 编辑区）。

### 3.2 启动与初始化
服务启动时自动：
- 创建所需目录（`data/`、`icons/`、`images/`）。
- 初始化数据库和表结构。
- 挂载静态资源目录与默认占位图。

## 4. 数据模型
### 4.1 `entries`
- `id` INTEGER PRIMARY KEY
- `keyword` TEXT NOT NULL
- `note_text` TEXT NOT NULL
- `icon_path` TEXT NULL
- `created_at` TEXT
- `updated_at` TEXT

### 4.2 `entry_images`
- `id` INTEGER PRIMARY KEY
- `entry_id` INTEGER NOT NULL
- `image_path` TEXT NOT NULL
- `sort_order` INTEGER DEFAULT 0

关系：`entries (1) -> (N) entry_images`。

## 5. API 设计
- `GET /`：返回网页。
- `GET /api/search?q=...`：按关键词搜索条目（`keyword` 与 `note_text`）。
- `POST /api/entries`：创建条目。
- `PUT /api/entries/:id`：更新条目。
- `DELETE /api/entries/:id`：删除条目及关联文件。
- `POST /api/upload/icon`：上传头像图标。
- `POST /api/upload/image`：上传正文图片。

## 6. 核心交互与数据流
### 6.1 创建
1. 前端上传图标/图片到对应上传接口。
2. 服务保存文件并返回相对路径。
3. 前端提交条目数据写入数据库。

### 6.2 搜索
- 输入触发搜索请求（可做轻量防抖）。
- 查询条件：
  - `keyword LIKE '%q%'`
  - `note_text LIKE '%q%'`
- 结果去重后返回条目列表。
- 列表显示图标；若未设置图标则展示默认占位图。

### 6.3 编辑
- 点击条目加载详情。
- 支持修改关键词、正文、替换图标、增删正文图片。
- 更新后刷新列表和详情。

### 6.4 删除
- 删除条目时同步清理数据库记录与关联文件。
- 清理失败不阻断主流程，但记录日志。

## 7. 前端页面结构（极简）
- 顶部：搜索框。
- 主区：
  - 结果列表（图标 + 关键词 + 摘要）
  - 详情编辑区（关键词、正文、图标上传、图片管理）

视觉风格：简洁、低装饰、可读性优先。

## 8. 错误处理与边界
- 上传仅允许图片类型（jpg/png/webp/gif）。
- **不设置图片大小上限**（按已确认需求）。
- `keyword` 必填并去空白。
- `id` 非法、记录不存在、参数缺失返回明确错误。
- 空搜索词时返回最近更新条目（前 N 条）。
- 仅监听 `127.0.0.1`。
- 删除前前端做二次确认。

## 9. 验收标准
- 可创建/编辑/删除条目，支持文本、可选图标、可选多图。
- 搜索可命中主关键词与正文关键词。
- 无图标条目可显示默认占位图。
- 服务重启后数据和图片仍可用。
- 异常请求不会导致服务崩溃。

## 10. 非目标（本期不做）
- 导入/导出备份。
- 高并发优化。
- 全文检索引擎（如 FTS）。
